macleanup

A free macOS cleanup CLI you can run with one command — and read line by line.

npx macleanup

macOS 11+·Apple Silicon + Intel·Source-available·No telemetry

01What it is

A single bash file (~2,400 lines) plus an 80-line Node launcher for the npx magic. Total install size: 41 KB tarball. No GUI. No subscription. No telemetry. No phone-home. The whole tool fits in less space than a single icon from a typical Electron app.

27 cleanup sections

From Xcode caches to orphaned LaunchAgents — covered below.

Real --dry-run

Every destructive op routes through helpers that no-op in dry mode. Period.

Persistent reports

Every run writes a dated .txt at ~/.mac-cleanup/reports/.

5 named profiles

dev, minimal, cache-only, deep, audit.

Zero network calls

Optional --check-update queries npm once. That's the entire footprint.

Multi-select UI

1,3,5-10 · all · none. Review before bulk delete.

03Why pick this over the alternatives

Most cleanup tools either delete too aggressively (and break things) or do too little (and leave gigabytes of cruft). macleanup sits in the middle.

Typical paid cleaners rm -rf blog posts macleanup
Confirms before deleting sometimes never always (per-section)
Tells you what each path is rarely no yes
Handles companion data on uninstall partly no yes (12 paths checked)
Skips Apple-managed caches yes no yes
Has a real --dry-run sometimes no yes
Source you can read no n/a yes
Costs money yes no no
"A tool that runs rm -rf and sudo on your behalf should be auditable.
Mine is one file you can read in your editor."

04Three ways to run

1. Zero-install via npx (easiest)

npm fetches the package into its cache, runs the interactive menu, reclaims its own cache afterwards. Your reports + logs persist at ~/.mac-cleanup/ — never inside the cache.

npx macleanup

# preview the safe-batch cleanup without touching anything
npx macleanup --dry-run --all

# run one specific section
npx macleanup --only 23 --stale-build-days 90 --dry-run

# show every section, then exit
npx macleanup --list

2. Global install via npm

Same package, persistent install. Both macleanup and mac-cleanup are registered as bin entries.

npm install -g macleanup
macleanup --version
macleanup --profile dev --dry-run

3. Direct git checkout (no Node required)

If you don't want Node in the loop, run the bash script directly.

git clone https://github.com/aoneahsan/macleanup.git
cd macleanup
chmod +x mac-cleanup.sh
./mac-cleanup.sh

05Where things live

Reports + logs persist regardless of which install method you pick. Override either path with --logs-dir or --reports-dir.

~/.mac-cleanup/
├── logs/
│   └── mac-cleanup-YYYY-MM-DD.log
└── reports/
    ├── orphans-YYYY-MM-DD.txt
    ├── unused-apps-YYYY-MM-DD.txt
    ├── large-files-YYYY-MM-DD.txt
    ├── stale-build-YYYY-MM-DD.txt
    ├── large-stale-YYYY-MM-DD.txt
    ├── launch-audit-YYYY-MM-DD.txt
    └── disk-usage-YYYY-MM-DD.txt

Each artefact starts with a credits header (tool version, author, repo URL, npm URL, run timestamp, host) so it's self-attributing. Add --no-reports to skip the .txt files, or --cleanup-logs-on-finish to delete the run's log on exit.
