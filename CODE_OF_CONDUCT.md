# Code of Conduct

This project is maintained by a single author. Interactions are expected
to stay short, respectful, and on-topic. The bar is low and the rules
are common-sense — but they are enforced.

## Expected behaviour

- **Be respectful.** Disagree about technical choices; don't disparage
  people. "I think this is wrong because X" is welcome. "You are wrong
  because you are X" is not.
- **Be specific.** When reporting bugs, include macOS version,
  `mac-cleanup --version`, the command you ran, and what you expected
  vs. what happened. Vague reports get closed with a request for
  details.
- **Be patient.** This is an unpaid side project. Response times vary
  from hours to weeks. Bumping a thread every day will not help.
- **Be honest.** Don't fabricate reproductions, screenshots, or claims
  about your environment.

## Unacceptable behaviour

- Personal attacks, slurs, or harassment of any kind.
- Sustained derailing of issue threads.
- Demanding fixes or features as if entitled to them. This project owes
  nothing to anyone.
- Posting other people's private information.
- Submitting AI-generated bug reports without verifying them yourself.
- Spamming PRs to inflate contribution counts.

## Scope

This Code of Conduct applies to all project spaces — GitHub issues,
discussions, pull requests, commit messages, the project email
(<aoneahsan@gmail.com>), and any other channel where the project name
or maintainer is invoked.

It also applies when an individual is officially representing the
project in public spaces.

## Enforcement

The author reserves the right to:

1. Edit or delete comments, commits, code, issues, PRs, and other
   contributions that violate this Code of Conduct.
2. Lock threads.
3. Block users from the GitHub repository, with or without warning.
4. Decline to engage with anyone who has been previously asked to stop
   a behaviour and continued.

Reports can be sent privately to <aoneahsan@gmail.com>. Reports will
be handled discreetly. There is no formal review board — final calls
are the maintainer's.
